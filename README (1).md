# Task Manager — CSC 4360 Mobile App Development

A Flutter application for managing tasks with real-time Firebase Firestore synchronization, full CRUD operations, and nested subtask support.

---

## Enhanced Features

### 1. Real-Time Search / Filter
A search bar at the top of the task list filters tasks instantly as the user types. The filter runs client-side on the already-streamed Firestore data, so no additional network calls are made. When no results match, a contextual empty-state message is displayed that includes the search term.

### 2. Delete Confirmation Dialog
Rather than immediately deleting a task, tapping the delete icon shows an `AlertDialog` asking the user to confirm. This prevents accidental data loss — a common UX concern in CRUD apps where deletions are irreversible. The dialog includes the task name so the user can verify which item will be removed.

## Setup Instructions

### Prerequisites
- Flutter SDK 3.x installed and on your PATH
- An Android emulator or physical Android device
- A Google account for Firebase

### Step 1 — Clone and Install Dependencies
```bash
git clone <your-repo-url>
cd task_manager
flutter pub get
```

### Step 2 — Create Firebase Project
1. Go to [console.firebase.google.com](https://console.firebase.google.com)
2. Click **Add project** → name it (e.g. `task-manager-csc4360`) → Continue
3. In **Firestore Database** → **Create database** → **Start in test mode**

### Step 3 — Connect Flutter to Firebase
```bash
# Install FlutterFire CLI (one-time)
dart pub global activate flutterfire_cli

# From inside the project folder:
flutterfire configure
```
Select your Firebase project when prompted. This generates `lib/firebase_options.dart` automatically.

### Step 4 — Run the App
```bash
flutter run
```

### Step 5 — Build APK
```bash
flutter build apk --release
# Output: build/app/outputs/flutter-apk/app-release.apk
```

## Known Limitations

- **No user authentication** — all tasks are stored in a shared Firestore collection. Any user with the app can see all tasks. Adding Firebase Auth would scope data per user.
- **No offline persistence** — Firestore offline caching is not explicitly enabled. Tasks will not load without a network connection. Enabling `PersistenceSettings(synchronizeTabs: true)` or `FirebaseFirestore.instance.settings` with offline persistence would fix this.
- **Subtask ordering** — subtasks are stored as an array and use `arrayUnion` / `arrayRemove`, which does not guarantee ordering on removal of duplicates. A subcollection approach would offer more robust subtask management at scale.
- **No pagination** — all tasks are loaded in a single stream. With thousands of tasks, this would degrade performance. Firestore `.limit()` and cursor-based pagination would be needed.
- **APK is debug-signed** — the release APK uses a debug keystore. For a real Play Store submission, a production keystore is required.
